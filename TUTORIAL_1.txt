=================================================== models/hotel.py ===================================================
1.0. O modelo é criado com os respectivos atributos e funções estáticas necessárias (ao final do arquivo)

1.1. O que é database_config(name)?
    -> Configuração do nome e formato do banco

1.2. O que é database_cursor(engine)?
    -> Configuração da var que receberá o cursor (validador de comandos SQL)

1.3. O que é database_init(engine)?
    -> Criação virtual do arquivo do banco no projeto, com seus respectivos atributos

1.4. O que é json(object_)?
    -> Essa função pega cada objeto do banco e os converte para exibição em formato "json"
    -> A sintaxe dos dados é "chave string de cada atributo: parâmetro.atributo chamado"
    -> Essa função é usada como auxiliar na função seguinte: "database_as_var(exec_)"

1.5. O que é database_as_var(exec_)?
    -> Essa função acessa o banco pelo cursor criado em "1.2" via função "query", onde o modelo é o parâmetro
    -> Se há algum objeto já criado, este será enviado a uma var dentro desta função, no formato da função em "1.4"
    -> Essa função é usada como auxiliar na função seguinte: "database_query(exec_)"

1.6. O que é database_query(exec_)?
    -> Essa função usa a função em "1.5" para obter os dados do banco
    -> Pode haver dados ou não, então essa função ou retorna uma lista de objetos "json" ou uma lista vazia

1.7. Onde essas funções são usadas?
    -> Essas funções são enviadas ao diretório do projeto: menu/1_query.py
